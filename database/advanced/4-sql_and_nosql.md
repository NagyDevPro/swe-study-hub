# Sql and NoSql


# Sql

- Structured Schema: Strongly typed tables and relationships make data integrity and complex joins straightforward.

## Pros
- data integrity
- high security


## Cons
- costy in joins
- can make failgures
- vertically scaling

# NoSql

- Not Only Sql: Sql Database but with some `violations` to native sql db rules

## Pros

- flexiable schema (documents, key-value)
- easy i/o, retrive of data
- supports horizontally scaling

## Cons

- No Data Integrity
- No Relations
